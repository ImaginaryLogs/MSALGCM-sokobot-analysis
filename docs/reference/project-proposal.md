# Research Question Formulation

## **Main Question**

_How can heuristic strength (tighter admissible heuristics), macro-graph tunnel abstraction precomputing, and heuristic weight tuning reduce the effective branching factor of PSPACE-complete pathfinding problems, and to what extent do these algorithmic shortcuts transfer from Sokoban to HP-Lattice Protein Folding?_

_(Revised 2026-07-18: state-space symmetry pruning replaced by heuristic strength — board symmetry
is rare in real Sokoban maps and yields near-null ratios; see [DECISIONS.md](../DECISIONS.md) #2.)_

| Supporting Question                                                      |                                                                                                                                                                                                        |
| ------------------------------------------------------------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Problem Domain:** What problem domain is addressed?                    | How can the 2D grid mechanics and box-pushing constraints of a Sokoban board be mapped to the 2D lattice structure and path constraints used in simplified protein-folding simulations?                |
| **Assumptions:** What assumptions does the approach make?                | What structural features must a graph search space possess for state-pruning, precomputing, and heuristic weight-tuning to effectively reduce exploration time?                                        |
| **Strengths and Weaknesses:** What are its strengths and weaknesses?     | Where does the extra runtime overhead of processing and filtering duplicate states outweigh the actual time saved during graph traversal?                                                              |
| **Complexity Characteristics:** What are its complexity characteristics? | Given that Sokoban is PSPACE-complete, how does our search efficiency ratio (total nodes expanded in our optimized search vs. a baseline search) change as map layouts become larger and more crowded? |
| **Scalability:** How scalable is the approach?                           | How scalable are three approaches? As the grid dimensions scale or the number of movable objects increases, how does the memory footprint and execution time change?                                   |
| **Practical Applications:** What practical applications exist?           | How can the algorithmic benchmarking results and pruning techniques developed on a video game testbed be applied to optimize simplified 2D biological search tasks like the HP-Lattice model?          |
